# 토목사업소 월드컵 결과 예측

대한민국 대 멕시코 경기의 예상 스코어에 여러 사람이 참여하고, 각자 입금
완료를 표시하는 모바일 대응 웹앱입니다.

## 먼저 시험하기

`index.html`을 브라우저에서 열면 별도 설정 없이 데모 모드로 실행됩니다.
데모 모드의 내용은 현재 브라우저에만 저장됩니다.

여러 사람이 동시에 사용하려면 아래의 Firebase 설정을 한 번 진행한 뒤
GitHub Pages에 올려야 합니다.

## 1. Firebase 공동 저장 설정

1. [Firebase Console](https://console.firebase.google.com/)에서 프로젝트를 만듭니다.
2. **빌드 > Authentication > 시작하기 > 로그인 방법**에서 **익명** 로그인을
   사용 설정합니다.
3. **빌드 > Realtime Database > 데이터베이스 만들기**를 선택합니다.
   위치는 가까운 `Singapore (asia-southeast1)`을 권장합니다.
4. Realtime Database의 **규칙** 탭에 `database.rules.json` 내용을 붙여넣고
   게시합니다.
5. **프로젝트 설정 > 내 앱 > 웹 앱 추가**에서 웹 앱을 등록합니다.
6. 표시된 `firebaseConfig` 값을 `firebase-config.js`의 각 항목에 넣습니다.
7. GitHub Pages 배포 후 Firebase의
   **Authentication > Settings > 승인된 도메인**에
   `사용자명.github.io`를 추가합니다.

Firebase 웹 설정값은 브라우저에서 사용하는 공개 식별자입니다. 비밀번호나
GitHub 개인 액세스 토큰을 이 파일에 넣으면 안 됩니다.

## 2. GitHub Pages 배포

1. GitHub에서 새 저장소를 만듭니다.
2. 이 폴더 안의 파일을 저장소 최상위에 올립니다.
3. 저장소의 **Settings > Pages**로 이동합니다.
4. **Build and deployment**에서 `Deploy from a branch`를 선택합니다.
5. Branch는 `main`, 폴더는 `/(root)`를 선택하고 저장합니다.
6. 잠시 뒤 표시되는 `https://사용자명.github.io/저장소명/` 주소를 부서에
   공유합니다.

## 동작 방식

- 점수는 대한민국 `3점 → 0점`, 멕시코 `0점 → 3점` 순으로 총 16개입니다.
- 같은 스코어에 여러 사람이 이름을 등록할 수 있으며 참여 입력창은 계속 유지됩니다.
- 같은 브라우저에서도 같은 스코어에 여러 이름을 연속 등록할 수 있습니다.
- 이름을 등록한 브라우저에서 본인 이름의 입금 상태만 변경할 수 있습니다.
- 각 이름 옆의 등록 취소 버튼으로 해당 등록 한 건을 취소할 수 있습니다.
- 입금 완료를 누르면 해당 이름과 상태가 초록색으로 바뀝니다.
- 원하는 점수가 없으면 대한민국·멕시코 득점을 입력해 행을 추가할 수 있습니다.
- 행 삭제 기능은 제공하지 않으며 Firebase 규칙에서도 행 삭제를 허용하지 않습니다.
- 초기화 버튼에 비밀번호 `12`를 입력하면 등록 이름, 입금 상태, 추가 행이
  모든 사용자 화면에서 초기화됩니다.
- Firebase 설정 전에는 로컬 데모 저장 방식으로 자동 전환됩니다.
